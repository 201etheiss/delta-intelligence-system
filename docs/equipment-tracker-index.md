## Overview

Multi-tier asset tracking platform for Delta360 field equipment. Tracks tankers, generators, pumps, compressors, trailers, and industrial assets across three operational tiers with GPS check-ins, geofenced sites, automated alerts, dispatch, compliance, and billing.

Fleet size as deployed: **7,613 assets** (6,842 active, 412 in maintenance, 359 rental).

Real branch yards (from deltaTanks.csv, 1,434 real records):
- Shreveport (1019)
- Midland (1020)
- Lake Charles (1022)
- Port Allen (1038)
- Corpus Christi (1039)

## Tracking Tiers

| Tier | Enum | Device Types | Description |
|------|------|-------------|-------------|
| Tier 1 | `TIER_1_FLEET` | GPS_SATELLITE, IoT | Fleet vehicles with satellite/IoT continuous tracking via Samsara |
| Tier 2 | `TIER_2_MOBILE` | GPS_CELLULAR, BLE_BEACON | Mobile equipment with cellular GPS polling |
| Tier 3 | `TIER_3_STATIC` | RFID, MANUAL_ONLY | Static assets tracked via RFID, BLE, or manual check-in |

Mobility types: `MOBILE`, `SEMI_MOBILE`, `STATIC` — drives alert severity and geofence logic.

## Data Model

### Core Entities

**Equipment** — central record. Key fields:
- `assetTag` — auto-generated D360-{TYPE_CODE}-{NNN} (e.g., D360-TNK-001)
- `trackingTier` — TIER_1_FLEET | TIER_2_MOBILE | TIER_3_STATIC
- `mobilityType` — MOBILE | SEMI_MOBILE | STATIC
- `status` — ACTIVE | MAINTENANCE | IDLE | RETIRED | IN_TRANSIT
- `type` — TANKER | GENERATOR | PUMP | COMPRESSOR | TRAILER | TOOL | OTHER
- Tank-specific: `tankType`, `tankSize`, `containmentType`, `productType`, `ownershipType`
- Lifecycle: `lifecycleStage` (PREFAB → SHOP_REPAIR → READY → IN_TRANSIT → SETUP → IN_SERVICE → MAINTENANCE → PICKUP → DECOMMISSION)
- Financial: `purchasePrice`, `depreciationMethod`, `salvageValue`, `usefulLifeYears`
- Fuel: `fuelCapacityGallons`, `currentFuelLevel`, `totalGallonsDelivered`
- Cross-system links: `samsaraVehicleId`, `fleetpandaId`, `salesforceAssetId`

**Site** — physical location with geofence boundary
- `radius` — geofence radius in meters (used for haversine breach detection)
- `type` — FIELD_SITE | OFFICE | YARD | WAREHOUSE | REMOTE

**Branch** — operational yard (Shreveport, Midland, Lake Charles, Port Allen, Corpus Christi)

**Customer** — end customer with Salesforce sync (`salesforceAccountId`, `salesforceData`)

**Device** — tracking hardware attached to equipment
- `deviceType` — GPS_SATELLITE | GPS_CELLULAR | RFID | BLE_BEACON | MANUAL_ONLY
- `trackingMethod` — SATELLITE | IOT | CELLULAR | RFID | BLE | MANUAL
- `batteryLevel`, `pingIntervalSeconds`, `lastPingAt`
- `samsaraVehicleId`, `samsaraGatewayId` — Samsara integration links

**Checkin** — GPS position event
- Sources: MANUAL | IOT | SATELLITE | CELLULAR | RFID | BLE | PROXIMITY | SAMSARA
- Fields: `latitude`, `longitude`, `altitude`, `accuracy`, `speed`, `heading`
- PostGIS GEOGRAPHY column auto-generated from lat/lng (raw SQL migration)

**Alert** — triggered anomaly
- Types: UNEXPECTED_MOVEMENT | GEOFENCE_BREACH | STALE_PING | LOW_BATTERY | STATUS_CHANGE | MISSED_CHECKIN | DEVICE_OFFLINE
- Severities: INFO | WARNING | CRITICAL
- Acknowledges via `acknowledgedById`, `acknowledgedAt`

**LocateRequest** — real-time "find this asset" request
- Status: PENDING | FULFILLED | TIMEOUT
- Used with Socket.io for live dispatcher-to-field-crew locate workflow

**RentalAgreement** — customer rental contract
- `status` — DRAFT | ACTIVE | EXPIRED | CANCELLED
- `billingFrequency` — WEEKLY | BIWEEKLY | MONTHLY | QUARTERLY | ANNUAL | ON_DELIVERY

**Ticket** — work orders and service requests
- Categories: MAINTENANCE | INSPECTION | DELIVERY | PICKUP | REPAIR | LEAK_REPORT | FILL_REQUEST | RELOCATION | COMPLIANCE | BILLING | OTHER
- AI fields: `aiSummary`, `aiCategory`, `aiPriority` (auto-generated by ticket-ai.ts)
- `TicketRule` model for automated ticket generation with cooldown logic

**Transfer** — equipment movement between branches/sites

**Route** / **RouteStop** — dispatch routes with sequenced stops (DELIVERY | PICKUP | INSPECTION | SERVICE | TRANSFER)

**Inspection** / **InspectionTemplate** — structured field inspections with JSON response capture

**Invoice** / **InvoiceLineItem** / **Payment** — billing, QuickBooks sync

**Document** — attachments (INSPECTION | DELIVERY_TICKET | PHOTO | PERMIT | INVOICE | CONTRACT)

**FuelTransaction** — fuel delivery/consumption/transfer/adjustment records

**Contract** / **ContractEquipment** — service and rental contract management (Phase 5)

**DepreciationEntry** — asset book value tracking

**Integration** — pluggable integration catalog with status tracking

### Geofences (PostGIS — raw SQL, not Prisma)

Stored in `geofences` table with PostGIS GEOMETRY columns. Two types:
- `INCLUSION` — alert if equipment is OUTSIDE the zone
- `EXCLUSION` — alert if equipment is INSIDE the zone

`checkPointInGeofences()` in `src/lib/geofence-utils.ts` runs raw SQL against PostGIS.

## Alert System

**Detection logic** (`src/lib/alerts.ts`):

1. **UNEXPECTED_MOVEMENT** — STATIC equipment moved >100m from last known position. Triggered on every new check-in against previous check-in. Severity: CRITICAL.

2. **GEOFENCE_BREACH (site radius)** — haversine distance from site center > `site.radius`. Severity: CRITICAL if STATIC, WARNING if MOBILE.

3. **GEOFENCE_BREACH (PostGIS)** — advanced polygon/circle geofences. INCLUSION type alerts when outside; EXCLUSION type alerts when inside. Per-geofence severity.

**Cooldown** (Samsara batch checks): 15-minute per-equipment-per-type cooldown via in-memory Map to prevent alert spam.

**Real-time delivery**: Socket.io via `emitAlertNew()` in `src/lib/socket.ts`. Clients subscribe to alert events via `useSocket.ts` hook.

## Integration Points

### Supabase
- Project ID: `uttmfowppfupdsacuhzz`
- Host: `aws-1-us-east-1.pooler.supabase.com:5432`
- Driver: `@prisma/adapter-pg` (PrismaPg) — NOT datasourceUrl pattern
- SSL: `{ rejectUnauthorized: false }` always
- PostGIS enabled for geofence polygon storage and point-in-polygon checks

### Samsara
- Real-time GPS positions for Tier 1 fleet vehicles
- Vehicle ID linking via `equipment.samsaraVehicleId` and `device.samsaraVehicleId`
- Polling triggers `checkSamsaraGeofences()` batch alert check after each cycle
- API handler in `src/lib/samsara/`

### Fleet Panda
- Fleet management integration via `fleetpandaId` on equipment records
- API handler in `src/lib/fleetpanda/`

### Mapbox
- Map style: `mapbox://styles/mapbox/dark-v11`
- Dynamic import for SSR compatibility
- HTML markers (not React components) for performance — updated imperatively via `marker.setLngLat()`
- Native clustering for high-density asset maps
- Geofence boundaries rendered as PostGIS polygon data

### Salesforce
- JWT Bearer auth flow (`SALESFORCE_JWT_PRIVATE_KEY`)
- Max 5 concurrent requests (rate limiter with queue)
- Stale data fallback on failure
- Fields: `customer.salesforceAccountId`, `equipment.salesforceAssetId`

## Key Architecture Details

### Real-time (Socket.io)
- Custom HTTP server in `server.ts` wraps Next.js with Socket.io
- `getIO()` singleton via `globalThis`
- Emit helpers in `src/lib/socket.ts`: `emitCheckinNew`, `emitAlertNew`, `emitEquipmentUpdated`, `emitAlertAcknowledged`, `emitLocateRequest`
- Client hook: `src/hooks/useSocket.ts` (global singleton)

### Auth (NextAuth.js v4)
- Credentials provider with bcrypt
- Rate limit: 5 failed attempts per email per 15 minutes
- Roles: FIELD | DISPATCHER | ADMIN | DRIVER | TECHNICIAN | MANAGER | BILLING | SAFETY | EXECUTIVE | CUSTOMER_PORTAL | READONLY | SUPER_ADMIN
- Field users: check-in + view. Dispatchers: manage equipment. Admins: full access.

### Prisma
- Client generated at `src/generated/prisma/client` (NOT `src/generated/prisma`)
- PrismaPg adapter pattern — no `datasourceUrl`
- Pool: max 1 connection in production, max 5 in dev

### Analytics Caching
- In-memory TTL cache (`src/lib/analytics-cache.ts`, 60-second default)
- Prevents repeated heavy aggregation queries on dashboard load

### Ticket Automation
- `TicketRule` model with `RuleTriggerType` conditions
- Cooldown hours per rule to prevent duplicate ticket spam
- AI triage via `src/lib/ticket-ai.ts` sets `aiSummary`, `aiCategory`, `aiPriority`
