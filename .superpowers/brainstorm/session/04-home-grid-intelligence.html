Intelligence Summary Anatomy

▶ Resume Where You Left Off

Tracks the user's last active module, page, and context (e.g., "reviewing AP aging for Magellan"). Stored as a lightweight session state object (module, page, scroll position, active filters, timestamp). One-click jump back. This is Nova's session continuity — it remembers what you were doing.

🔄 What Changed Overnight

Aggregates events since last session: new invoices, completed deliveries, automation runs, data anomalies. Sourced from the Event Store once it's live — for now, pulled from gateway API diffs and automation logs. Grouped by category with counts. Clickable to drill into each change.

⚠ Needs Your Attention

Action items surfaced by Nova: overdue invoices, reconciliation exceptions, closing opportunities, flagged anomalies. Each links directly to the relevant module/page. Badge count ("7 action items") persists on the Home tile in the Rail. This replaces the old dashboard's static KPI cards with dynamic, personalized intelligence.

📈 Activity Timeline

Horizontal strip of recent platform events — automation completions, anomaly detections, delivery confirmations, alert triggers. Real-time feed from the event stream. Clicking any item opens the relevant module. Timestamps are relative ("2h ago"). Filterable by source (automations, feeds, alerts).

Module Grid Personalization

Per-User Configuration

Pinned modules float to top. Module order is draggable. Group visibility — users can hide modules they don't use. View mode — Grid (tiles) or List (compact rows). All stored in user preferences (existing user-preferences.ts). Admins can set role-based defaults (accountants see Finance pinned, ops sees Fleet pinned).

Dynamic Groupings (Future)

Module groups are data-driven, not hardcoded. Admin can create/rename/reorder groups. Users can override group assignments for their view. Groups can be role-scoped (HR group only visible to HR role). For now: 7 fixed groups matching the current nav structure. Schema supports arbitrary groups from day one so migration is config, not code.
