Shell Components

▲ Nova Bar ALWAYS VISIBLE

Persistent AI intelligence strip across the top. Nova input field for natural language queries from anywhere. Proactive briefing badge ("3 items need attention"). Notification bell. Density mode toggle (EX/OP). User avatar. This never goes away — it's the OS-level AI presence.

◀ Status Rail ALWAYS VISIBLE

Thin icon bar on the left (VS Code activity bar pattern). Module group icons for quick navigation. Service health dots at the bottom (green/yellow/red for each data source). Chat and Search are always accessible from the rail.

▶ Module Tabs MULTI-MODULE

Browser-tab pattern for open modules. Multiple modules open simultaneously. Close individual tabs. "+" to open a new module (returns to home grid). Active tab highlighted with orange underline. This replaces the current sidebar navigation.

▼ Workspace MODULE CONTENT

The main content area where module pages render. This is where all 62 existing DI pages will live. Each page is wrapped as a module within the workspace. The workspace respects the current density mode (Executive/Operator).

Key Design Decisions to Confirm

1. Navigation Model

Current: Traditional sidebar with 7 collapsible groups + 50 nav items
Proposed: Status Rail (icons only) + Module Grid home + Module Tabs (open pages as tabs)
Effect: The sidebar goes away entirely. Navigation is: click a module tile on home grid, or click a rail icon to jump to that module group. Individual pages within a module use SubNavTabs (already exists in DI).

2. How Existing Pages Map

All 62 pages are grouped into ~7 modules (Finance, Operations, Intelligence, Compliance, Organization, Platform, Admin). Within each module, pages are accessible via the existing SubNavTabs component. So: Rail icon → Module Tab opens → SubNavTabs for page switching within the module. Same pages, new container.

3. Incremental Approach

Build the shell (Nova Bar + Rail + Tabs + Workspace) as a new layout wrapper. Mount existing pages inside it. No page rewrites needed initially — just re-nesting. Then iterate: density modes, Nova integration, module grid polish.
