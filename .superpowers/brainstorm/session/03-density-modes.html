Density Modes — Content-Aware

Same data, different presentations. Executive sees insight. Operator sees detail.

Each module renders differently based on density mode. Executive shows curated KPIs, progress bars, AI insights, and clean whitespace. Operator shows dense tables, monospace numbers, inline controls, and maximum information density. Toggle on Nova Bar applies globally. Each page implements both views.

Density Mode Pattern — How Pages Implement It

1. React Context Provider

A DensityProvider wraps the shell. Exposes useDensity() hook. Returns "executive" or "operator". Toggle on Nova Bar updates context. Role determines default (managers→EX, accountants→OP). User override persists to localStorage.

2. Page-Level Rendering

Each page reads density from context and renders accordingly. Executive renders KPI cards, progress bars, AI insight banners, summary charts. Operator renders full data tables, monospace numbers, inline controls, section headers, drill-down buttons.

3. Shared Components

Common density-aware components: DensityKPI (card vs inline strip), DensityTable (summary vs full), DensityChart (large visual vs sparkline). Pages compose from these — consistent look without per-page CSS work.
