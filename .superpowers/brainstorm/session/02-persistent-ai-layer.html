How the Persistent AI Layer Works

▲ Nova Bar (Top Strip)
  • Always visible, never navigated away from
  • Quick-invoke input — type to Nova from anywhere
  • Alert pills — click to see briefings, active bots, running automations
  • Density toggle (Executive/Operator) affects all module content
  • Notification bell with inbox dropdown
  • Nova avatar click → full Nova command palette
◀ Chat Panel (Slide-Out)
  • Toggle from Rail chat icon — slides over workspace
  • Context-aware: knows which module/page you're viewing
  • Module context chips show what data Nova can access
  • Three modes: Chat, Bots (saved automations), History
  • Suggested actions with one-click bot execution
  • Conversation persists across module switches
⚡ Automations & Bots (OS-Level)
  • "Bots Active" pill on Nova Bar — always visible count
  • Bots tab in chat panel — manage from anywhere
  • Bot execution happens in background, results in chat
  • Scheduled automations run via cron — existing engine
  • Automations page still exists as a full module for config
  • Quick actions: "Run AP Aging Audit" inline in chat
🏠 Module Awareness
  • Nova Bar input updates: "currently viewing [module]"
  • Chat panel header shows active module context
  • Context chips auto-populate from current page
  • Nova's system prompt includes module schema + vocabulary
  • Switching modules updates context — conversation continues
  • Existing /chat page becomes the "full-screen chat" option

What Changes vs. Current DI

Current: Chat is a Page

Right now /chat is a full page you navigate to. You leave your current view. The assistant and automations are separate pages too (/assistant, /automations). Context is lost when you switch.

DataOS: Chat is an OS Layer

Chat becomes a slide-out panel accessible from anywhere via the Rail. Nova knows what you're looking at. The /chat page still exists for dedicated deep conversations, but the panel is the primary interaction. Bots and automations surface through the Nova Bar pills and the chat panel's Bots tab.
